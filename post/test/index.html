Cookie和Session的一些理解

时间： 2018-03-11 分类： web   php   字数： 1232 字 阅读： 3分钟 阅读次数：

概要

  • HTTP是无状态，无法记住状态。
  • 该如何才能记住登录状态
  • 通过cookie和session实现
  • 谷歌浏览器通过以下连接确认cookie的信息
    chrome://settings/siteData

Cookie　基本

  • 从web服务器通过响应头发送给web浏览器
    通过「键=值」的方式设置。
  • web客户端请求web服务器时，web服务器通过请求头携带的cookie，识别对方身份
  • Cookie每次连接服务器都会发送
  • Cookie依赖域名而存在，访问某个服务器时，只会携带目标的服务器的cookie，其他服务器的cookie并不会被发送
    // setcookie(名称,値,过期时间,目录,域名,是否安全,是否只允许服务器读取);
    //
    //• 目录，目标目录下才会生效
    //• 域名，该域名下有效
    //• 是否安全，如果为true，则https的请求时才有效（SSL）
    //• 是否仅限服务器读取，如果为true，则前端javascript读取不到
    
    // 添加cookie
    setcookie('short-userid','ralph','/','exaple.com');
    
    // 删除cookie
    setcookie('short-userid','');
  • 简单使用指定到第三个参数就行，后面的参数有默认值。
  • 第三个参数时时间戳的形式。(1970年1月1日开始的秒数)
  • 通过time()函数，可以直接获取到秒数。
    例)两周后过期　(14日、24时、60分、60秒)
    setcookie(‘my_id’, $my_id, time() + 60 * 60 * 24 * 14);
  • setcookie 如果只设置了第一个参数，那么等于设置该cookie过期的操作。
  • Cookie php通过「$_COOKIE」可以取到cookie。
  • setcookie 在页面返回前声明好。
  • Cookie 的域名一般不用设置。
    <?php
    if (isset($_COOKIE['my_id'])){
        $myId = $_COOKIE['my_id'];
    }else{
        $myId = '';
    }
    ?>

Session　基本

  • 声明周期为，浏览器关闭之前
  • 即使连接跳走后，也不会丢失
  • session_unset()删除全部的session
  • session保存的内容，以文件的形式保存在服务器端。
  • 客户端的cookie中的只保存 sessionId
  • 服务器端会根据cookie带来的sessionId自动识别对方身份。

php.ini設定

session.use_cookies

这个值如果设置为0，那么就会禁用掉cookie。
所以请一直将它的值设成 1。

    session.use_cookies = 1
    session.use_only_cookies = 1

session.auto_start

每个页面的最开始都需要使用session_start()开启session，可以将此值设为1，就不需要每个页面都声明了。

    session.auto_start = 1

session.cookie_httponly

HttpOnly设置的情况

    session.cookie_httponly = on
    session.entropy_file & session.entropy_length

需要生成安全的sessionId的情况

    [Session]
    ;;entropy_file windows的环境不需要设置
    session.entropy_file = /dev/urandom
    session.entropy_length = 32
    
    /dev/urandom是Linux和UNIX系统用来生成随机数的。

session.cookie_secure

session.cookie_secure设置的情况

    session.cookie_secure = On
