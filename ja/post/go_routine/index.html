gorutine 其实是一种协程 Coroutine

gorutine 的定义

  • 任何函数只需要加上go就能送给调度器运行
  • 不需要在定义时区分是否是异步函数
  • 调度器在合适的点切换，和线程的切换是不一样的
  • 使用-race来检测数据访问冲突
  • 轻量级的“线程”
  • 非抢占式的 多任务处理，由协程主动交出控制权
  • 编译器/解释器/虚拟机层面的多任务，并不是操作系统层面的，操作系统层面是没有协程概念的
  • 多个协程可能在一个或者多个线程上运行
  • main函数本身也是个 go routine

    // 以下代码会死机，因为以下的go rutine不会主动的交出控制权
    func main() {
        var a[10] int
        for i := 0; i < 10; i ++ {
            go func(i) {  // 这个i感觉不用传，但是必须要传，
                a[i] ++
                // 手动交出控制权，所以不会死机，不过这里只是演示而已
                // 因为实际我们总是会有机会进行调度的切换的
                runtime.Gosched() 
            }(i)
        }
        
        time.Sleep(time.Millisecond)
    }
    
    // 以下代码会不会死机，但是打印的内容并不是连贯的，所以感觉他是抢占式的
    // `fmt.Println`属于io操作
    func main() {
        for i := 0; i < 100; i ++ {
            go func(i) {
                fmt.Println(i)
            }(i)
        }
        time.Sleep(time.Millisecond)
    }

gorutine 可能的切换点

  • 以下只是参考，不能保证切换，不能保证在其他地方不会切换
  • i/o,select
  • channel
  • 等待锁
  • 函数调用（有时）
  • runtime.Gosched()
